# Скилл: Сценарий для видео

## Когда использовать
Клиент говорит: «сценарий», «напиши сценарий», «сделай сценарий по видео», «нужен сценарий на тему...», «хочу записать ролик про...», «скринкаст про...», «разбери ролик».

## Два режима

### Режим 1: Сценарий по видео (из чужого ролика)
Клиент скидывает ссылку на YouTube или текст транскрипции.
Результат: полный анализ + сценарий для съёмки своего ролика.

### Режим 2: Сценарий по теме (с нуля)
Клиент описывает тему и формат. Примеры:
- «Сценарий на тему: как установить скиллы для Claude Code, скринкаст, 10 мин»
- «Хочу снять ролик про 5 способов монетизировать AI, talking head, 15 мин»
Результат: готовый сценарий для съёмки.

## Процесс — Режим 1 (по видео)

### Шаг 1: Транскрипция
- Если клиент скинул текст — использовать его
- Если ссылка на YouTube — скачать аудио через yt-dlp, транскрибировать через Deepgram
- Если транскрипция уже есть в кэше бота — спросить, есть ли текст

### Шаг 2: Анализ + сценарий
Генерирую по промпту из ~/projects/transcriber-bot/konspekt/prompts/scenario.txt
Результат содержит 6 блоков:
1. Хук — дословно из оригинала (для изучения)
2. Open Loops — все крючки с адаптацией под воронку клиента
3. Структура контента — блоки с таймкодами
4. CTA — призывы к действию
5. Мета — название, длительность, пропорции
6. Готовый сценарий — шпаргалка для съёмки

### Шаг 3: Выдача
HTML-файл или текстом в чат.

## Процесс — Режим 2 (по теме)

### Шаг 1: Уточнить
- Тема ролика
- Формат: talking head / скринкаст / микс
- Хронометраж: 5 / 10 / 15 / 20 / 30 мин
- Дополнительные пожелания (если есть)

### Шаг 2: Генерация
Сценарий с нуля по промпту из ~/projects/transcriber-bot/konspekt/prompts/scenario_topic.txt

### Шаг 3: Выдача
HTML-файл или текст в чат.

## Формат сценария (для обоих режимов)

Каждый блок:

⏱ БЛОК: [Название] — [X мин]

Ключевые смыслы (что ОБЯЗАТЕЛЬНО сказать):
• Смысл 1 — короткая фраза
• Смысл 2 — короткая фраза

Опорные точки (порядок мыслей):
1. Начни с [тема] → [фраза]
2. Потом [тема] → [фраза]
3. Закончи [тема] → [фраза]

🪝 Open loop (врезка после блока):
«Точный текст для произнесения на камеру»
→ Куда: Telegram / бот / курс

🔗 Переход к следующему блоку:
«Связующая фраза»

⚠ Адаптировать: [что заменить на своё]

## Правила

### Хук в сценарии
- Максимум 4-5 опорных точек
- Главные идеи и фразы-триггеры
- Шпаргалка, не текст для зачитывания — клиент снимает БЕЗ суфлёра

### Хронометраж
- У каждого блока ориентир в минутах
- Сумма ≈ заданная длительность ролика

### Open Loops
- Конкретные фразы для произнесения на камеру
- Привязаны к блокам: «после блока X скажи: ...»
- Адаптированы под воронку клиента (Telegram {{CHANNEL}}, боты, курсы)

### Переходы
- Между каждым блоком — связующая фраза

### Ключевые смыслы
- 2-4 мысли в каждом блоке, которые ОБЯЗАТЕЛЬНО сказать

### Адаптация
- ⚠ помечать где заменить на своё: кейсы, ссылки, цифры, имена
- НЕ оставлять чужие продукты без пометки

### Стиль
- Использовать стиль из knowledge/writing-style.md и knowledge/post-structure.md
- Регистр 1 (Энергичный наставник): «ребят», «короче», «давайте», цифры, кейсы
- Фирменные словечки: «вот смотрите», «блин, а что если», «и самое главное»
- Контрасты: «раньше X — сейчас Y»
- Пол автора: мужской

## Итерации
Клиент может попросить:
- «Переделай хук» — переписать вступление
- «Добавь свои open loops» — вставить крючки под конкретную воронку
- «Подробнее блок про X» — развернуть конкретный раздел
- «Сделай короче / длиннее» — изменить хронометраж
- «Добавь скринкаст-инструкцию» — пошаговые действия на экране
- «В моём стиле» — усилить авторский голос из writing-style.md
